AB TESTING

For a lovely plant shop UI




Part 1: Creating Hypothesis


Introduction

To enhance user experience and drive traffic, many websites conduct regular A/B tests to find out which changes to the websites are valuable and are worth investing. Among its competitors, plant shop cact-I + cact-you = cact-us is developing a more user friendly UI and needs AB testing to see which version of the interface performs better.


To achieve that, I designed version A and version B, which is demonstrated below:

Version A

Version B

The greatest difference between the two versions lies mainly in font-size and text-hierarchy. Version B is designed as the controlled group while A adds some independent variables. A has larger font-size, clearer text-hierarchy (price, description, and more information if user is interested) that are marked by differing font-weight and font-size. By these independent variables, I make my hypothesis.



Creating Hypothesis


Time to Completion


Null Hypothesis: the time to completion wouldn't be much different with the price of the cactus is highlighted in bold-face and put in a more conspicuous location.

Alternative Hypothesis: the time to completion would be shorter with the price of the cactus is highlighted in bold-face and put in a more conspicuous location. (time A < time B)

Reasoning: One of the major differences between the two versions is the highlight of prices of the cactuses in bold-face and heavier font-weight. In this case, it is expected that it takes shorter time to find the cactus that's at the price level that suits buyer's interest. Different cactuses in the highlighted price version are also contained in separate cards to show the contrast between different products.


Return Rate


Null Hypothesis: A better text-hierarchy through different font-size wouldn't cause more users to return to the site.

Alternative Hypothesis: More users would NOT return to the page when there is a better text-hierarchy in the shopping page.

Reasoning: I am trying to reject the null hypothesis and prove the alternative hypothesis.The reasoning that there will be more users who wouldn't return to the site when there's a better text-hierarchy is that, when the price tag has bigger font-size and different information fall under their own text-hierarchy, the sorted information will make it easier for users to estimate the total price and make their purchases without coming back and double checking the information.





Part 2: Data Collection and Wrangling


Data Collection Process

To collect data, I posted my deployed Heroku link to the studio spreadsheet for peer data collection. I also send the link to some friends and gave them instructions to gather data from users who are less familiar with the goal of the experiment. Please refer to Infographic for more detail on the user components of this experiment.


Note on Data Cleansing Process

To clean the myfilteredlog.txt data, I transformed the txt file into a more readable csv file. I deleted the users who did not make any clicks and only opened the website. Due to Heroku's limitation on the lines of log it can keep track of, I got rid of the initial data that are part of the previous user's record. Some users use the website multiple times in different tabs, I recorded each instance of interaction with the interface as a separate data point to increase the sample size. Time to Completion is calculated by subtracting timestamps (page load time - click time) between first opening the website and closing the website. Number of returns is calculated by whether the user returned to the website or not, instead of how many times a user returns to the site.

Part 4: Conclusion and Takeaways



For time to completion, A performs much better than B. Since p-value is greater than alpha = 0.05, we can reject the null hypothesis. Thus, having a bold-face price tag can accelerate user's purchasing process and reduce the time for completion.

For the return rate, since p-value is greater than alpha = 0.05, we can reject the null hypothesis. Thus, having a clearer text-hierarchy can cause more users to not return to the shopping page. However, we do not know exactly the specific implication of a lower return rate, since due to the small sample size, it is also likely that some users did not make purchase or is not returning to the site anymore. The implication would be clearer if there is more data suggesting user behavior.

More broadly, AB Testing for this assignment really informs me the importance of having a larger sample size to get more accurate results. Apparently, the p-values for both t-test and chi2-test is relatively high compared to what could be with a larger sample size with users having different levels of familiarity with the website. It is also pretty interesting how changes in minute details can actually change the user experience.
